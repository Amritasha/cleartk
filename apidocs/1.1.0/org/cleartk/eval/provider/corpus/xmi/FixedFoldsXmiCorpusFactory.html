This extension of XmiCorpusFactory assumes that the folds for a corpus are fixed. The folds are specified files that contain lists of file names (one file name per line.) So, for example, getTrainFile() might return a file named "/train.txt" that contains a list of those XMI files that belong in the training data set.

Please see CorpusReaderProvider for a description, broadly speaking, of how you should set up your corpus. Here, this is implemented by simply creating files that contain file names for the different sets and folds. The names of the files that you should create might be:

  • fold-1.txt - consists of the xmi file names for the test set for fold 1
  • fold-2.txt
  • ...
  • fold-n.txt
  • test.txt - consists of the xmi file names for the test (i.e. holdout evaluation) set.
  • train.txt - consists of the xmi file names for the training set
For the list above, the method getFoldFiles() would return the paths for the files "fold-1.txt" ... "fold-n.txt", getTrainFile() would return the path to the file "train.txt" and getTestFile() would return the path to the file "test.txt".
